Introduction to Turing.jl

A quick introduction to probabilistic programming language Turing.jl

Turing.jl is a probabilistic proramming language that let’s us define a generative model and does inference.

1 Coin example

Let’s flip a biased coin a hundred times.

using Turing
using DataFrames
coin_flips = rand(Bernoulli(0.7), 100);

In order to do inference, we need to specify a model.

\[ p \sim Beta(1,1) \]

\[ coin flip \sim Bernoulli(p) \]

@model function coin(data)
  p ~ Beta(1,1)
  for i in eachindex(data) 
    data[i] ~ Bernoulli(p)
  end
end
coin (generic function with 2 methods)

1.5 Chains

The results of sampling runs are MCMCChains.Chains instances.

A Chains instance has the following fields:

  • value: AxisArray object :iter x :var x :chain
  • logevidence
  • name_map (to define sections, NamedTuple section -> names, default section is :parameters)
  • info
value = rand(500, 2, 3)
chn = Chains(value, [:a, :b])
chn2 = Chains(value, ["A[1]", "A[2]"])

One can restrict the output (an AxisArray) to some parameters by using chn[:a], for example. To get a Chains instance with only parameter a, do chn[[:a]]. Weird!

An AxisArray has fields :axes and :data. It has advanced indexing. I.e. to get only a slice of a dimension :dim, one can use A[dim = idx], so in the case of a MCMC sample, we might use chn.value[var = :a], or chn.value[chain = 2]

summarystats(chn; sections = :parameters)

quantile(chn; sections = :parameters)

or describe(chn; sections = :parameters) to get both at once

Turings sample returns a Chains instance with sections :parameters and :internals.

(parameters = [:p], internals = [:lp, :n_steps, :is_accept, :acceptance_rate, :log_density, :hamiltonian_energy, :hamiltonian_energy_error, :max_hamiltonian_energy_error, :tree_depth, :numerical_error, :step_size, :nom_step_size])

DataFrame(posterior[posterior.name_map.parameters])
